Fix critical bug: Pass provider/model from overview.md to marker and unifier agents

CRITICAL BUG FIX: Marker and unifier agents were always using Claude Code regardless
of the configured provider in overview.md. This caused users to hit Claude Code quota
limits even when using Codex or Gemini.

## The Bug

**Root cause:**
- `mark_structured.sh` and `mark_freeform.sh` load `DEFAULT_PROVIDER` and `DEFAULT_MODEL` from `overview.md`
- BUT when generating marker and unifier task commands, these values were never passed to the agents
- Result: Agents used their hardcoded default (`default="claude"` in marker.py line 122)
- Users configured Codex/Gemini but were unknowingly calling Claude Code

**Evidence:**
- User configured: `default_provider: codex` or `default_provider: gemini`
- Error messages showed: "/upgrade to max" and "/extra-usage" (Claude Code specific)
- All 196+ tasks were actually calling Claude Code, not Codex/Gemini
- User's Claude Code quota exhausted even though they thought they were using Codex

## The Fix

### marker.py and unifier.py Task Generation

**mark_structured.sh:**
- Line 248: Added `--provider '$DEFAULT_PROVIDER' ${DEFAULT_MODEL:+--model '$DEFAULT_MODEL'}` to marker tasks
- Line 362: Added `--provider '$DEFAULT_PROVIDER' ${DEFAULT_MODEL:+--model '$DEFAULT_MODEL'}` to unifier tasks

**mark_freeform.sh:**
- Line 223: Added `--provider '$DEFAULT_PROVIDER' ${DEFAULT_MODEL:+--model '$DEFAULT_MODEL'}` to marker tasks
- Line 339: Added `--provider '$DEFAULT_PROVIDER' ${DEFAULT_MODEL:+--model '$DEFAULT_MODEL'}` to unifier tasks

### Other Agents (Already Correct)

These agents were already receiving provider/model arguments:
- ✓ pattern_designer.py (lines 209-210 in mark_structured.sh)
- ✓ normalizer.py (lines 273-274 in mark_freeform.sh)
- ✓ aggregator.py (lines 414-415 in mark_structured.sh)

The bug only affected parallel tasks (marker and unifier).

## Before Fix

```bash
# User configures:
default_provider: codex
default_model: gpt-5.1

# Generated task (WRONG):
python3 marker.py --activity A1 --student 'John' --submission '...' --output '...'
# Uses default="claude" from marker.py line 122

# Result:
# - Claude Code called instead of Codex
# - Codex quota untouched
# - Claude Code quota exhausted
# - Error: "/upgrade to max" (Claude Code message)
```

## After Fix

```bash
# User configures:
default_provider: codex
default_model: gpt-5.1

# Generated task (CORRECT):
python3 marker.py --activity A1 --student 'John' --submission '...' --output '...' --provider 'codex' --model 'gpt-5.1'

# Result:
# - Codex called correctly
# - Claude Code untouched
# - Codex quota used
# - Error (if quota hit): Codex-specific messages
```
